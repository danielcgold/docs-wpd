<h2><span class="mw-headline" id="Site_Structure">Site Structure</span></h2>
<p>The root of the wiki is <code>docs.webplatform.org/wiki/</code>. This is the wiki where all documentation will exist, and it is collaborative.
</p><p>The structure of the wiki is broken down into technology, characteristic, and article name. For example, the URL for the <code>href</code> attribute in HTML is <code><a href="/wiki/html/attributes/href" title="html/attributes/href">html/attributes/href</a></code>. 
</p><p>Article names are case-sensitive (e.g. <code>getAttribute</code>). but the proposal is to use lowercase names for the hierarchy (e.g. <i>html</i> instead of <i>HTML</i>). 
</p>

<h2><span class="mw-headline" id="Page_Types">Page Types</span></h2>
<p>These are types of reference pages we need for the site's information architecture.
</p><p><b>Note:</b> These are basic primitives that need unique templates in MediaWiki; having a unique template allows us to perform queries and aggregations, and supplements the page hierarchy architecture structure as a method of categorization; another way of categorizing them is through MediaWiki categories that can be added to each page. <h3><span class="mw-headline" id="Notes">Notes</span></h3>
<ul><li> Should attributes in the same language that apply to different elements with different values for each have a different page, or should there be an aggregate page with multiple sections?</li></ul>
<h2><span class="mw-headline" id="Notes_2">Notes</span></h2>
<ul><li> We want aggregated pages, where content from other related pages is transcluded into a single page
<ul><li> '<b>Example:'</b> the page for an interface will contain all the transcluded information from the individual method and property pages</li>
<li> '<b>Example:'</b> the page for the <b>font</b> shorthand property will also contain the pages (or links to) font-family, font-feature-settings, font-kerning, font-language-override, font-size, font-size-adjust, font-stretch, font-style, font-variant, font-variant-ligatures, and font-weight properties</li>
<li> '<b>Example:'</b> the page for an element will contain all the information for each of its attributes, which are each a separate page. (<b>Note:</b> what about the <b>href</b> attribute, which appears on multiple elements... maybe some thing like a sub-form, for context for each element where it appears?)</li></ul></li></ul>
<ul><li> For CSS property values (and other pages), we want not only a linked <i>data type</i> (e.g. &lt;length&gt;), but also an explanation of what the length means in that context, the range of values, etc. (Some sort of sub-form? see "Multiple-instance templates")</li></ul>
