<h1><span class="mw-headline" id="Verbatims">Verbatims</span></h1>
<p>The table below lists the raw comments from participants at the Berlin (Feb. 8 &amp; 9) and San Francisco (Feb. 23) doc sprints, and tracks the bugs logged against these issues. The gathering of this information, along with the <a href="/wiki/WPD:Community/Survey" title="WPD:Community/Survey">survey</a> was the first effort toward gaining insight into the usability of the wiki for contributors. Herein are summarized the findings with a bias toward raising issues with respect to editing and contributing.
</p>
<h2><span class="mw-headline" id="Summary">Summary</span></h2>
<p>The verbatim information below is summarized here by category. Not all categories are summarized.
</p>
<h3><span class="mw-headline" id="Design.2FLayout">Design/Layout</span></h3>
<ul><li> Broken, unimplemented features (Comments, Q&amp;A) are confusing users.</li>
<li> Users are unable to log in because the log-in elements do not appear on the home page.</li></ul>
<h3><span class="mw-headline" id="Editing_tasks">Editing tasks</span></h3>
<ul><li> Editing in MediaWiki and with SMW forms is not easy (this runs contrary to one of our basic assumptions)</li>
<li> Session bugs are a serious impediment to productivity.</li>
<li> Need a guide to creating examples.</li>
<li> Need a process (and a flag) for reviewing contributions.</li>
<li> Topics and topic clusters are confusing and rife with abuse.</li>
<li> How to documentation (WPD:*) is hard to find and difficult to follow.</li>
<li> Markdown is preferred over MediaWiki markup.</li>
<li> Flag design is scaring some users away from working on pages.</li></ul>
<h3><span class="mw-headline" id="Forms">Forms</span></h3>
<ul><li> Syntax section too restrictive, not applicable to all cases.</li>
<li> (Many other issues, see below.)</li></ul>
<h3><span class="mw-headline" id="Getting_Started">Getting Started</span></h3>
<ul><li> Need to identify tasks according to role, domain expertise, skill.</li>
<li> Need tasks for designers, and others.</li>
<li> Supplement the Getting Started pages with video.</li>
<li> Need a more obvious path to getting started from home pages.</li></ul>
<h3><span class="mw-headline" id="Search">Search</span></h3>
<ul><li> Duplicate pages in search results - one for each instance of the search term (this is low-hanging; why is it still on the tree?)</li>
<li> Can't search internal help pages.</li></ul>
<h2><span class="mw-headline" id="Next_steps">Next steps</span></h2>
<p>To help with the resolutions to these problems, in the list below cite the relevant bug from our project management system. If the issue requires a new bug, please create it.
</p>
